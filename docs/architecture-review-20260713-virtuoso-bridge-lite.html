After

Remote Execution

execute · upload · download · identity

唯一 interface
SSH adapter / local adapter / test adapter

ControlMaster、超时预算、平台差异留在 implementation 内

Problem

调用者知道 SSHRunner 具体属性与返回形状，传输知识跨 seam 泄漏。

Solution

建立面向远程执行能力的窄 interface，由 SSH 与本地 adapter 实现。

Wins
  • • locality：超时集中
  • • leverage：多个调用域复用
  • • 测试只跨一个 interface

Problem

内部 module 反向导入公开包根，形成循环初始化风险并扩大测试面。

Solution

领域逻辑仅依赖最小执行 interface；VirtuosoClient 成为 adapter。

Wins
  • • seam 可被真实测试 adapter 替换
  • • locality：执行契约集中
  • • 包初始化方向单向

GDS Export State Machine

stage → launch → observe log/artifact → classify → publish → cleanup

预算、恢复、local/remote 差异全部内聚

Problem

导出可靠性很强，但配置与状态分支使 interface 接近 implementation 复杂度。

Solution

以请求/结果收窄 interface，把状态转换与 adapter 选择留在 implementation。

Wins
  • • leverage：可靠导出一处复用
  • • locality：失败分类集中
  • • 表驱动测试覆盖状态

Top recommendation

深化远程执行 module

它跨越 Virtuoso、Maestro、Layout 与 Spectre，能用一个更窄的 interface 同时提升 locality、测试替换能力和跨平台可靠性。
