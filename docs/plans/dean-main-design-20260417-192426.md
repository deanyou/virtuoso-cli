## Problem Statement

模拟电路设计工程师现在用手工方式跑仿真：设置 → 仿真 → 看结果 → 调参 → 重复。这个迭代循环极其费时间。每个bandgap/LDO/OTA IP都要反复人工介入才能收敛。

## What Makes This Cool

把LLM的推理能力和gm/Id方法论结合起来，让机器自动完成"读结果 → 想对策 → 调参数 → 再仿真"的循环。工程师只需要确认机器给出的候选电路。等于给每个模拟工程师配备了一个永不停歇的EE助手。

## Target User & Narrowest Wedge

**用户:** 芯片模拟电路设计工程师（多人通过服务器共享）

**首发IP:** Bandgap（电压基准，通用性强）

**最小可行场景:** 输入spec（目标电压、PSRR、TC等）→ LLM驱动仿真调参 → 生成候选电路 + 报告 → 工程师确认

## Constraints

- 多用户服务器部署（Linux服务器，工程师远程连接）
- Virtuoso通过virtuoso-cli（Rust TCP daemon + SKILL bridge）接入
- 仿真结果自动判定（spec比对pass/fail）
- 需要存档（工程师自确认 + review给老板看）
- AI能力依赖gm/Id方法论的形式化

## Premises

1. gm/Id方法论可以形式化地指导晶体管sizing，并且这个过程可以被程序化 — **agreed**
2. Virtuoso的SKILL接口足够稳定，能支撑批量自动化仿真循环 — **agreed**
3. 工程师接受"机器生成候选 + 人工确认"的工作流 — **agreed（信任锚点是人工确认这一步）**

## Approaches Considered

### Approach A: 批量自动化（最小可行）
Summary: 输入参数范围 + 仿真设置（YAML），自动跑完所有参数组合，输出结果表格
Effort: S | Risk: Low | Pros: 立即可用 | Cons: 仍需人工判断下一步

### Approach B: AI辅助调参
Summary: LLM读仿真结果 → 用gm/Id方法论生成下一个参数候选 → 循环收敛
Effort: M | Risk: Med | Pros: 减少人工迭代轮次 | Cons: 需要稳定prompt + 结果解析

### Approach C: 端到端自动优化（推荐）
Summary: 输入spec → LLM全权调参 → 自动收敛 → 生成候选电路报告 → 工程师确认
Effort: L | Risk: High | Pros: 最接近"哇"的效果 | Cons: 黑盒风险大，工程师可能不信任
**Note: 工程师不会完全信任纯机器输出。但"机器生成候选 + 人工确认"是信任锚点，不是负担。生成电路后手动确认也很快速。**

## Recommended Approach

**Approach C — 端到端自动优化 + 人工确认**

架构分层：

```
用户（工程师）
  ↓ spec输入（YAML）
virtuoso-cli（Rust CLI）
  ↓ TCP daemon
Virtuoso（SKILL bridge）
  ↓
Spectre仿真
  ↓
LLM推理层（gm/Id规则引擎 + 可插拔模型）
  ↓ 循环收敛（max_iter=20, Δspec<1%）
候选电路（Spectre netlist + 参数JSON）
  ↓
工程师确认（关键信任锚点）
  ↓
仿真报告存档（Markdown + 数据表格）
```

**收敛判定标准：**
- 连续2次迭代，spec偏差Δ<1% 视为收敛
- max_iter=20 防止无限循环
- 仿真超时（单次>30min）自动跳过该候选并记录

**输出格式：**
- 候选电路：Spectre netlist（可直接加载进Virtuoso）+ 参数JSON
- 报告：Markdown文件，包含spec对比表、迭代轨迹、可视化曲线

**首发实现路径：**
1. **多用户队列支持**（服务器场景，day one requirement）
2. virtuoso-cli增加batch仿真命令（输入参数范围 → 自动扫参 → 输出表格）
3. 增加spec YAML解析和pass/fail判定
4. 增加LLM推理层：读仿真结果 → 用gm/Id规则生成下一个参数候选
5. 增加循环收敛逻辑
6. 增加报告生成（Markdown + 仿真数据）

## Success Criteria

- 一个工程师能在30分钟内，用自然语言输入spec，让机器给出bandgap的最优候选
- 候选电路经过人工确认后能直接用于后续流程
- 报告能够存档并在review时展示给老板
