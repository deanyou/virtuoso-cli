## New Modules to Test

### 1. `src/spectre/batch.rs` — BatchSimulator

**What it does:** Generates parameterized netlists from a template, runs multiple spectre jobs, aggregates results.

**Test file:** `src/spectre/batch.rs` (add `#[cfg(test)] mod batch_tests`)

**Test cases to write:**

| Test | Description |
|------|-------------|
| `test_empty_param_range` | When param ranges are empty, return empty netlist vector |
| `test_single_param_combo` | Single W/L value produces exactly 1 netlist |
| `test_template_substitution` | Template with `${W}` gets replaced with actual values |
| `test_template_missing_param` | Missing template param produces an error |
| `test_all_jobs_succeed` | 3 jobs all succeed → summarize returns all results |
| `test_partial_failure` | 2/3 jobs fail → summarize returns failures with error messages |
| `test_all_jobs_fail` | All jobs fail → summarize returns error + empty data |
| `test_job_timeout` | Simulate timeout (mock or short timeout) → recorded as error |

### 2. `src/optimizer/mod.rs` — Optimizer

**What it does:** Takes simulation results, checks convergence (Δ<1%), generates next param candidate, manages iteration history.

**Test file:** `src/optimizer/mod.rs` (add `#[cfg(test)]` module)

**Test cases:**

| Test | Description |
|------|-------------|
| `test_convergence_delta_1pct` | Two consecutive iterations with Δ<1% → returns converged |
| `test_convergence_not_met` | Δ>1% → returns not converged |
| `test_convergence_empty_history` | No history → returns not converged |
| `test_convergence_max_iter` | 20 iterations reached → returns converged (max reached) |
| `test_best_effort_returns_closest` | All fail but one is closest → returns that one |
| `test_best_effort_empty_history` | No history → returns error |
| `test_candidate_generation` | Given failed results, generates new W/L values in range |

**Note:** `test_candidate_generation` needs gm/Id table. If table doesn't exist yet, mock it or skip until table format is confirmed.

### 3. `src/queue/mod.rs` — JobQueue

**What it does:** Multi-user async job queue with enqueue/dequeue/complete operations.

**Test file:** `src/queue/mod.rs` (add `#[cfg(test)]` module)

**Test cases:**

| Test | Description |
|------|-------------|
| `test_enqueue_single_job` | One job enqueued → queue length = 1 |
| `test_dequeue_fifo` | Two jobs enqueued (A then B) → dequeue returns A first |
| `test_dequeue_empty` | Dequeue on empty queue → returns None |
| `test_complete_updates_status` | Job completed → status changes to Done |
| `test_complete_unknown_job` | Complete non-existent job → returns Error |
| `test_concurrent_enqueue` | Two tasks enqueue simultaneously → no race (use std::sync::Mutex) |

**Note:** Use `std::sync::Mutex` for internal queue state to support concurrent access.

### 4. `src/spec/bandgap.rs` — BandgapSpec

**What it does:** Parses YAML bandgap spec files into structured BandgapSpec.

**Test file:** `src/spec/bandgap.rs` (add `#[cfg(test)]` module)

**Test cases:**

| Test | Description |
|------|-------------|
| `test_valid_bandgap_yaml` | Valid YAML with all fields → parses to BandgapSpec |
| `test_missing_vbg_target` | YAML missing Vbg target → returns ParseError |
| `test_negative_w_or_l` | W=-1 or L=-1 → returns ValidationError |
| `test_default_corner` | No corner specified → defaults to "tt" |
| `test_param_range_order` | min > max → returns ValidationError |

**YAML example for tests:**
```yaml
ip_type: bandgap
target:
  Vbg: 1.20
  PSRR: 80  # dB
  TC: 20    # ppm/C
params:
  W:
    min: 1e-6
    max: 10e-6
    step: 1e-6
  L:
    min: 0.18e-6
    max: 1e-6
    step: 0.18e-6
corner: tt
```

## Failure Mode Gaps (Critical)

| Gap | Failure | User Impact |
|-----|---------|-------------|
| No timeout in batch | Stuck spectre job never completes | Batch hangs forever |
| No disk space check | Large PSF files fill /tmp | Silent failure on download |
| No SSH connection retry | Single SSH failure = entire job fails | Poor reliability on flaky networks |
| gm/Id table missing | Optimizer can't generate candidates | LLM fallback has no data |

All 4 gaps have no tests and no error handling. Add to TODOs.md.
